# Sécurité

Merci de ne pas publier de vulnérabilité sensible dans une issue publique si elle expose des données personnelles, des clés API ou une base SQLite.

## Données à protéger

FoodNote peut contenir :

- historique alimentaire ;
- poids et objectifs ;
- exports personnels ;
- base SQLite ;
- clés API, notamment Groq ;
- éventuelles informations MQTT/Home Assistant.

Ces données ne doivent pas être commités dans GitHub.

## Signaler un problème

Ouvrez une issue GitHub si le problème n'expose pas de secret. Pour un problème sensible, contactez le mainteneur par un canal privé si disponible.
